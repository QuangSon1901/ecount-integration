Overview

THG-FULFILL Open API allows partners to create shipping orders (Express & POD), track shipments, and receive real-time notifications through webhooks.

Base URL

https://express.thgfulfill.com/api/v1

Key Features

  • Create bulk Express shipping orders (up to 100 per request)
  • Create bulk POD (Print-on-Demand) orders for production & fulfillment
  • Query order status and tracking (works for both Express & POD)
  • Real-time webhook notifications for status changes and tracking updates
  • OAuth2 Client Credentials authentication
  • Reference data APIs for services, warehouses, and shipping methods
  • Sandbox environment for testing

Sandbox Mode

Sandbox accounts create real orders on ECount using the fortest customer code, allowing you to test the full flow without affecting production data. Both Express and POD orders use the same sandbox mechanism. Contact your account manager to get sandbox credentials.

Response Envelope

All responses use JSON with a consistent envelope:

{
  "success": true,
  "message": "...",
  "data": { ... },
  "timestamp": "2025-01-15T10:30:00.000Z"
}

HTTP Status Codes

CodeMeaning
200Success
201Created (all orders succeeded)
207Multi-Status (partial success in bulk)
400Bad Request — invalid or missing parameters
401Unauthorized — invalid or expired token
403Forbidden — insufficient permissions
404Not Found
429Too Many Requests — rate limit exceeded
500Internal Server Error

Authentication

The API uses OAuth2 Client Credentials flow. You'll need your client_id and client_secret from the dashboard.

1. Get Access Token

POST /api/v1/auth/token

Request Body

ParameterTypeRequiredDescription
client_idstringRequiredYour client ID (starts with thg_)
client_secretstringRequiredYour client secret
 
curl -X POST https://express.thgfulfill.com/api/v1/auth/token \
  -H "Content-Type: application/json" \
  -d '{
    "client_id": "thg_your_client_id",
    "client_secret": "your_client_secret"
  }'
const response = await fetch('https://express.thgfulfill.com/api/v1/auth/token', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    client_id: 'thg_your_client_id',
    client_secret: 'your_client_secret'
  })
});
const data = await response.json();
console.log(data.data.access_token);
$ch = curl_init('https://express.thgfulfill.com/api/v1/auth/token');
curl_setopt_array($ch, [
    CURLOPT_POST => true,
    CURLOPT_HTTPHEADER => ['Content-Type: application/json'],
    CURLOPT_POSTFIELDS => json_encode([
        'client_id' => 'thg_your_client_id',
        'client_secret' => 'your_client_secret',
    ]),
    CURLOPT_RETURNTRANSFER => true,
]);
$response = json_decode(curl_exec($ch), true);
$accessToken = $response['data']['access_token'];

Response

{
  "success": true,
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIs...",
    "refresh_token": "rf_abc123...",
    "token_type": "Bearer",
    "expires_in": 3600
  }
}

2. Use the Token

Include the access token in the Authorization header for all API requests:

Authorization: Bearer eyJhbGciOiJIUzI1NiIs...

3. Refresh Token

POST /api/v1/auth/refresh

Use the refresh token to get a new access token without re-authenticating.

4. Verify Token

GET /api/v1/auth/verify

Check if the current access token is still valid.

5. Revoke Token

POST /api/v1/auth/revoke

Revoke the current access token.

Create POD Orders

POST /api/v1/orders/pod/bulk

Create one or more Print-on-Demand orders. Maximum 100 orders per request.

Requires bulk_order_enabled permission. Uses the same authentication as Express orders.

Request Body

FieldTypeDescription
orderNumberstringRequiredYour unique order reference
warehouseCodestringRequiredWarehouse code — see POD Reference Data
shippingMethodstringRequiredShipping method — see POD Reference Data
receiverobjectRequiredReceiver info: name, countryCode, province, city, addressLine1, zipCode, phone (required), addressLine2, email (optional)
itemsarrayRequiredProduct items (min 1) — see below
trackingobject*Required for SBTT. Optional for others. Contains trackingNumber and linkPrint
customFieldsobjectOptionalCustom ECount field overrides

items Item

FieldTypeDescription
skustringRequiredProduct SKU code
namestringRequiredProduct name
quantityintegerRequiredQuantity (min 1)
pricenumberRequiredUnit price
productSizestringOptionalSize (e.g. S, M, L, XL)
productColorstringOptionalColor name
designUrlsarrayRequiredDesign files: [{ key: "front", value: "https://..." }]
mockupUrlstringRequiredMockup image URL

Webhooks

Receive real-time notifications when events occur on your orders (both Express and POD).

Available Events

EventTrigger
order.statusOrder status changed (e.g. pending → shipped)
tracking.updatedTracking number or shipping label received
order.exceptionAn exception occurred (delivery failed, customs hold, etc.)

Register Webhook

POST /api/v1/webhooks
FieldTypeDescription
urlstringRequiredYour HTTPS endpoint URL
secretstringRequiredSecret for HMAC-SHA256 signing (min 8 chars)
eventsarrayRequiredEvents to subscribe to

Payload Signature

Each webhook delivery includes these headers:

HeaderDescription
X-Webhook-EventEvent type (e.g. tracking.updated)
X-Webhook-SignatureHMAC-SHA256 of the full JSON body using your secret

Verifying Signatures

const crypto = require('crypto');

function verifyWebhook(payload, signature, secret) {
  const expected = crypto
    .createHmac('sha256', secret)
    .update(JSON.stringify(payload))
    .digest('hex');
  return signature === expected;
}
function verifyWebhook($payload, $signature, $secret) {
    $expected = hash_hmac('sha256', json_encode($payload), $secret);
    return hash_equals($expected, $signature);
}

Retry Policy

  • Up to 6 attempts with exponential backoff
  • Webhook is auto-deactivated after 5 consecutive failures
  • Successful delivery resets the failure counter

Error Codes

All error responses follow the same envelope format:

{
    "success": false,
    "message": "Validation failed for some orders",
    "data": {
        "summary": { "total": 1, "valid": 0, "invalid": 1 },
        "validationErrors": [{
            "orderIndex": 0,
            "orderNumber": "POD-2025-001",
            "errors": [{
                "field": "tracking.trackingNumber",
                "message": "trackingNumber is required for SBTT (Ship by TikTok)",
                "type": "any.required"
            }]
        }]
    },
    "timestamp": "2025-01-15T10:30:00.000Z"
}
Rate Limits: Each customer has hourly and daily rate limits. When exceeded, you'll receive a 429 response with a Retry-After header.
